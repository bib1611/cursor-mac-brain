# RUN — the daily loop (read this FIRST, every session)

This turns the Writing Brain from a library you *consult* into a machine you *run*. The brain knows how to write. This file decides what to write today, who it is for, and what it sells. Load it, then execute the chain in order. Do not skip a step.

**The one-line job:** produce SKU-tagged content that copies Adam's own proven winners, aimed at acute reader pain, in Adam's voice, that passes the anti-slop gate and the confrontation governor.

---

## THE CHAIN (run top to bottom)

### 0. Load the gates (non-negotiable, every output)
- `WRITING-BRAIN.md` PART 0 (voice) + PART 1 (anti-slop 8-dim gate).
- These never get skipped. A draft that fails PART 1 gets killed and rewritten, not shipped.

### 1. Read what already worked (this is the edge)
- `winners.md` — Adam's own validated posts. Pattern-match today's task to the closest winner and reuse the mechanic.
- `losers.md` — the kill-list. Do not repeat a dead angle.
- If both are still empty, fall back to PART 3 hooks — but theory is the floor. The goal is to fill these so the brain runs on Adam's truth, not a stranger's.

### 2. Load the revenue spine (what the writing is FOR)
- `WRITING-BRAIN.md` PART 14 — the SKU ladder and routing rules.
- Hold the question in front of every draft: **which SKU does this ladder to?** Pick ONE. Match register to brand (sword → Biblical Man, scar tissue → Dead Hidden + store).
- Check for any TIME-GATED SKU live right now (e.g. a weekend Vault offer). If one is closing, today's strongest CTA points there with internal urgency.

### 3. Find today's conversation (timely beats evergreen)
- What is the audience already thinking about today? Scan the Following feed chronologically, then For You (PART 12 drill). Bookmark anything reusable.
- Check the calendar: Sunday is the personal piece + highest sales day. Friday is the numbers piece.
- Hook-first: do not commit to a topic until a viral-shaped hook strikes. Often write the hook before the body.

### 4. Draft the day's output
Default daily set (PART 9 cadence, under-10k rules do not apply — Adam is at 52.9k):
- **2-3 original X posts**, spaced 3-4 hours, strongest first thing.
- **1 question/engagement post** (pulls replies — the #1 algo signal).
- **1 Substack Note** (Notes follow = free email sub; farm it like 2020 Twitter).
- **Reply-bait built into at least one post** (a confrontation or question, governed by the Confrontation Governor in PART 2).
- Each tagged with its SKU and its self-reply link.

**Pick the engagement style (PART 15) by goal:**
- X tweet for REPLIES/reach → STYLE A (Ed): hard verdict + a genuinely debatable, Scripture-defensible claim. Run the Confrontation Governor.
- Note or story-post for FREE-SUB conversion → STYLE B (Dakota): scene cold-open + a curiosity-loop subhead. The dwell pulls the sub.
- Substack post that must hook cold AND convert → Dakota cold-open, Adam confession-to-authority body, one Ed verdict line near the close, then the SKU CTA.
- Default: Dakota for Notes, Ed for tweets. Always fused with PART 0 voice, scored on PART 1, tagged to a SKU.

### 5. Run the gates on every draft before it ships
- [ ] TIER 1 floor: PART 1 anti-slop score — 7+ every dimension, 60+ total. Below 5 on AI-slop = auto-kill. Fail here and it is dead, no further grading.
- [ ] TIER 2 win-grade: run `GRADE.md` — declare the goal (replies / free-sub / sale), score /100, fix the weakest high-weight dimension. Under 55 = rebuild the hook before it ships.
- [ ] VOICE (blunt Dakota-register notes/tweets only): run `GRADE-DAKOTA.md`. Name the rung (1 clean / 2 Ed / 3 soft / 4 pass). Every box must pass. A soft box means it is stuck on rung 3 — rewrite to blunt before it ships.
- [ ] Voice check (PART 0): sounds like a 44-year-old former garbage truck driver at 4 AM.
- [ ] SKU field filled (PART 14): this ladders to ___. Not "none" unless deliberately brand-only.
- [ ] If it names/confronts: clear all 3 of the Confrontation Governor (PART 2).
- [ ] Link is in the self-reply, NOT the body. Substack uses `r=2t2o3r`, store uses `deadhidden.org/store/<slug>`.
- [ ] One CTA, not two.

### 6. Close the loop (this is what makes tomorrow smarter)
- At the end of the day OR start of the next, log yesterday's results:
  - Anything that cleared ~1k likes / 20k impressions / a sale → add a row to `winners.md`, tagged with the 2-3 `GRADE.md` dimensions it maxed (this is the seed the Tier 3 reweight reads).
  - Anything that died flat or drew blocks with no rally → add a row to `losers.md`.
- This single habit compounds. In 90 days the brain is tuned to Adam specifically and beats the entire masterclass archive.
