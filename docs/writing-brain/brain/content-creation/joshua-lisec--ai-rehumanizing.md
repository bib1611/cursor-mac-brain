## Core thesis
Use AI to speed writing without lowering quality. AI-smooth copy raises a "wall of disinterest and distrust" in the reader. Copy-pasting raw model output signals the writer has not grappled with the subject enough to know if it is even true, which is fatal to a discernment audience. The job is not to hide AI use, it is to edit AI back to human.

## The 6.5 AI tells (named or demonstrated)
1. **Negation framing.** "Not X, but Y" / "not just X, but Y." The sentence defines the topic by what it is not. Fix: delete the negated setup, lead with the affirmative claim.
2. **Word-density filler.** Inflated phrasing, a long clause where two plain words work. Fix: compress hard (Lisec's mechanical prompt was to tighten by ~85 percent when the model rambles).
3. **Vague abstraction.** Broad, moralized labels with no concrete referent. Fix: replace with names, dates, objects, mechanisms, consequences.
4. **Rule-of-three overuse.** Automatic triads because "three feels complete." Fix: keep three only when there are truly three, otherwise two, four, or asymmetrical.
5. **Metaphor / analogy as argument.** Persuading by "this is like that," or mixed metaphors. Fix: argue the subject directly; keep a metaphor only if precise and not carrying the claim.
6. **Prompt-answer opening.** Reads like an answer to a question only the prompt writer saw. Fix: make the first line stand alone for a cold reader (object, claim, reason to care).
6.5. **Surface tics.** Overused em dashes, colons, exclamation points, staccato fragments, emojis, adverb stuffing. Fix: normalize punctuation, cut decorative emphasis, keep an adverb only when the verb pairing is exact.

## Word-by-word re-humanizing workflow
1. Start from owned source material (notes, transcripts, bullets, quotes, citations, voice dumps). Avoid blank-prompt invention unless low-stakes or you can fact-check every claim.
2. Light-edit prompt only: ask for a very light readability edit. Add constraints: keep raw tone, keep original language, preserve the voice, do not sanitize the strongest lines.
3. If it returns word salad, force compression with a numeric target (~85 percent cut). Treat as rough.
4. First manual pass: delete the tells (strip "not X but Y," remove repeated triads, cut filler that adds no fact/image/turn).
5. Second pass: add precision. Ask of each abstract word, what exact thing/person/date/action/consequence. Say names, not groups.
6. Third pass: repair logic and sequence. Each sentence follows the last. Add context if it began mid-answer. Check that pairings and list items correspond.
7. Fourth pass: metaphor discipline. Delete metaphors that substitute for proof; remove mixed metaphors; keep paired concepts in one semantic field.
8. Fifth pass: restore a human judgment. Add the non-obvious point, sharp distinction, lived observation, or punchline a generic model would not supply. This is where text becomes authored.
9. Final pass: surface polish. Reduce decorative punctuation, remove emojis, keep fragments only for intentional rhythm, keep adverbs only when selected and correctly paired.

## Editing gate (reject or revise any sentence that)
- Tells the reader what the subject is not before saying what it is.
- Sounds like it is answering an invisible prompt.
- Uses three items because three sounds polished.
- Uses many words without adding factual calories.
- Relies on a metaphor instead of a claim.
- Avoids concrete names, dates, objects, or mechanisms.
- Uses punctuation as decoration.
- Sounds clean but contains no human judgment.
